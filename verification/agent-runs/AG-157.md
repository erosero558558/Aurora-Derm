# AG-157 - Close public V6 internal navigation state sync

## Scope

- Turn V6 internal navigation into a real reading aid instead of static anchor lists.
- Sync active state across page menu panels, hub catalog rail, internal rails, and legal route tabs.
- Preserve current V6 layout and `data-v6-*` contracts while improving orientation inside hub, service, telemedicine, and legal surfaces.

## Implementation

1. Marked section-navigation links explicitly with `data-v6-section-link`.
2. Marked page-menu and rail containers with `data-v6-section-nav` for predictable runtime targeting.
3. Added `bootSectionNavigation()` in `js/public-v6-shell.js`.
4. The runtime now:
   - groups all in-page section links by hash target,
   - computes the active section from scroll position and hash,
   - keeps `is-active` + `aria-current="location"` synchronized,
   - updates state immediately on click and on resize/hash/scroll.
5. Added `aria-current="page"` to the active legal policy tab.
6. Added visual active states for:
   - page menu links,
   - hub catalog rail pills,
   - internal rail links,
   - legal tabs.
7. Added `scroll-margin-top` to anchored sections so sticky chrome no longer clips destination sections.
