# AG-089 - Frontend hardening: alinear version de script.js en paginas de servicios

## Resumen

Se cerro un bloque grande de hardening frontend para eliminar drift de versiones de assets entre `index.html`, paginas de servicios y `sw.js`.

Cambios aplicados:

- Sincronizacion de `script.js?v=...` en `servicios/acne.html` y `servicios/laser.html`.
- Actualizacion de `sw.js` para usar la misma version de `script.js` y mantener alineados `bootstrap-inline-engine` y `styles-deferred`.
- Script nuevo `bin/sync-frontend-asset-versions.js` para sincronizar versiones automaticamente desde `index.html`.
- Scripts npm nuevos:
    - `npm run assets:versions:sync`
    - `npm run assets:versions:check`

## Archivos principales

- `bin/sync-frontend-asset-versions.js`
- `package.json`
- `servicios/acne.html`
- `servicios/laser.html`
- `sw.js`

## Validacion local

1. `npm run assets:versions:sync` -> OK
2. `npm run assets:versions:check` -> OK
3. `npx eslint bin/sync-frontend-asset-versions.js` -> OK
4. `npm run test:frontend:qa:closeout` -> OK (`44 passed` public, `29 passed + 1 skipped` admin, `18 passed` closeout subset)

## Resultado

Contrato de versiones frontend unificado y automatizado. Se elimina la falla reproducible de `tests/version-consistency.spec.js` y queda guardrail operativo para futuros bumps de version.
