# AG-085 - Frontend admin availability draft + bulk actions

## Resumen

Se implemento un bloque grande de UX para disponibilidad en admin:

- Flujo de borrador (dirty state) con `Guardar cambios` y `Descartar`.
- Acciones masivas de 7 dias (`duplicar` y `limpiar`) ademas de acciones por dia.
- Atajos de teclado para acciones nuevas.
- Ajustes visuales del panel y soporte dark mode.
- Actualizacion de pruebas Playwright de disponibilidad.

## Archivos principales

- `src/apps/admin/modules/availability.js`
- `src/apps/admin/index.js`
- `admin.html`
- `admin.css`
- `tests/admin-availability-responsive.spec.js`

## Validacion local

2. `npx playwright test tests/admin-availability-responsive.spec.js tests/admin-availability-readonly.spec.js` -> OK (7 passed)

4. `node agent-orchestrator.js board doctor --json` -> OK

## Resultado

Bloque admin availability cerrado con cambios funcionales, UX visible y cobertura de regresion de disponibilidad actualizada.
