# AG-150 - Align telemedicine production smoke with v3 markup

Date: 2026-03-02
Owner: Ernesto
Executor: codex

## Goal

Fix the telemedicine smoke spec after production validation showed selector drift caused by the V3 markup.

## Root cause

The production page already used:

- `data-escalation-model` with `service-timeline-v3__steps`
- a booking CTA inside the booking bridge scoped by `data-cta-target="booking"`

The previous spec still expected:

- `.service-timeline-v2__steps article`
- a generic `a[data-analytics-event="open_public_cta"]` inside the booking bridge, which became ambiguous

## Change

Updated `tests/telemedicina-conversion-qa.spec.js` to:

- assert `3` articles directly under `data-escalation-model`
- click the booking CTA using `a[data-cta-target="booking"]`

## Validation

```powershell
$env:TEST_BASE_URL='https://pielarmonia.com'
npx playwright test tests/telemedicina-conversion-qa.spec.js --workers=1
```

Result:

- `3 passed`

## Conclusion

The telemedicine smoke spec now reflects the V3 production markup and passes against the live domain.
