# AG-132 — Turnero sala XIV: onboarding operativo estrella con checklist persistente

Fecha de cierre: 2026-02-28  
Owner: Ernesto  
Executor: codex

## Objetivo

Agregar onboarding operativo persistente en el módulo de turnero admin para reducir errores de primera operación por estación.

## Cambios implementados

1. `admin.html`

- Se añadió botón `Reabrir guía` (`data-action="queue-open-onboarding"`) dentro del panel de control de estación.
- Se reforzó el panel `#queueOnboardingPanel` con:
    - cabecera visual,
    - indicador de progreso (`#queueOnboardingProgressPill`),
    - meta descriptiva (`#queueOnboardingMeta`),
    - checklist renderizable (`#queueOnboardingChecklist`).
- Se conservó el flujo de práctica (`Iniciar práctica`) y cierre (`Entendido`).

2. `src/apps/admin/index.js`

- Se agregó estado persistente de progreso del onboarding con `localStorage`:
    - `queueOnboardingProgressV2`
    - normalización/lectura/escritura robusta.
- Se incorporó checklist de 3 pasos:
    - `station_locked`
    - `shortcuts_opened`
    - `practice_completed`
- Se implementaron utilidades para:
    - sincronizar UI de checklist/progreso,
    - marcar pasos completados por eventos reales,
    - persistir progreso y restaurarlo tras recarga.
- Se añadió acción de reapertura de guía (`queue-open-onboarding`).
- Se ajustó el comportamiento de onboarding:
    - iniciar práctica no marca `seen` automáticamente,
    - cerrar guía con pasos incompletos pide confirmación.
- Se emiten eventos operativos del onboarding vía `piel:queue-ops` al completar pasos.

3. Build

- Se regeneró bundle admin (`admin.js`) con `npm run build`.

- La guía inicial del turnero ahora es persistente y medible por estación.
