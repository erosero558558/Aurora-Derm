## Summary

This block migrated the active public smoke and acceptance suite from legacy public assumptions to the real V6 surface contract.

The work also fixed a real V6 routing defect: the locale switch only swapped `/es/` and `/en/` roots and did not translate section paths such as `/es/servicios/` to `/en/services/` or `/es/telemedicina/` to `/en/telemedicine/`.

## What changed

1. Replaced legacy selectors such as `#citas`, `#appointmentForm`, `#serviceSelect`, `data-stage-*`, and old V3 shell markers in the active public suite.
2. Added V6 helper coverage for:
    - home runtime readiness
    - locale switch discovery
    - negative assertions against legacy public shell elements
3. Updated home, telemedicine, service, legal, mobile, navigation, deferred fallback, hydration, language, taxonomy, and staging acceptance tests to assert V6 DOM contracts.
4. Fixed locale-switch path mapping in `public-v6.js` for:
    - `/es/servicios/` <-> `/en/services/`
    - `/es/servicios/:slug/` <-> `/en/services/:slug/`
    - `/es/telemedicina/` <-> `/en/telemedicine/`
5. Rebuilt static V6 artifacts so git-sync deploy output matches the corrected source behavior.
