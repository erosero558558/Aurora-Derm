# AG-229 - Canonizar mocks de admin-auth en specs admin v3

- Fecha: 2026-03-13
- Owner: Ernesto
- Executor: codex

## Resumen

Se creo un helper compartido para mocks autenticados de `admin-auth.php` y se
migro la familia de specs `admin-v3`/`quick-nav` a la forma canonica del
contrato auth. Los mocks ya no responden solo `authenticated/csrfToken`; ahora
incluyen tambien `status`, `mode`, `configured`, `recommendedMode` y
`twoFactorEnabled`, alineados con el gate admin actual.

## Archivos

- `tests/helpers/admin-auth-mocks.js`
- `tests/admin-v3-shell.spec.js`
- `tests/admin-v3-canary-runtime.spec.js`
- `tests/admin-v3-visual.spec.js`
- `tests/admin-quick-nav.spec.js`

## Verificacion

- `npx eslint tests/helpers/admin-auth-mocks.js tests/admin-v3-shell.spec.js tests/admin-v3-canary-runtime.spec.js tests/admin-v3-visual.spec.js tests/admin-quick-nav.spec.js`
    - Resultado: limpio, sin errores ni warnings nuevos.
- `$env:TEST_LOCAL_SERVER_PORT='8047'; npx playwright test tests/admin-v3-shell.spec.js tests/admin-v3-canary-runtime.spec.js tests/admin-v3-visual.spec.js tests/admin-quick-nav.spec.js --workers=1`
    - Resultado: `9 passed`

## Resultado

Quedo reducida la deuda latente de mocks auth incompletos en la familia
`admin-v3`, y el siguiente barrido puede reutilizar el helper en otros specs
admin sin duplicar payloads legacy.
