# AG-227 - Realign admin Playwright smoke to current login gate modes

## Summary

- Realigned the login smoke coverage in `tests/admin.spec.js` to the current admin gate instead of relying on generic password/submit selectors.
- Made `setupLoginAdminMocks()` explicit and stateful for `legacy_password`, so the mocked login flow no longer falls back into the runtime-preferred OpenClaw mode.
- Updated the empty-password and invalid-password tests to assert the canonical login feedback card (`#adminLoginStatusTitle`, `#adminLoginStatusMessage`) and the canonical controls (`#adminPassword`, `#loginBtn`).

## Reproduced issue

```powershell
$env:TEST_LOCAL_SERVER_PORT='8042'; npx playwright test tests/admin.spec.js --grep login --workers=1
```

Observed failures before the fix:

- `login con contrasena vacia no funciona`
- `login con contrasena incorrecta muestra error`
- `login con 2FA muestra etapa dedicada y permite volver al paso de clave`
- `login exitoso actualiza el estado de sesion en el chrome v2`

Root cause:

- The tests assumed the visible gate was always the legacy password form.
- The mock helper returned `status` without `mode`, so the runtime defaulted back to the OpenClaw login surface and hid `#adminPassword`.

## Verification

```powershell
npx eslint tests/admin.spec.js
$env:TEST_LOCAL_SERVER_PORT='8043'; npx playwright test tests/admin.spec.js --grep login --workers=1
```

## Outcome

- `6/6` login-focused Playwright tests passing.
- ESLint clean.
- The admin smoke now validates the current canonical login contract instead of depending on brittle generic selectors or environment-preferred auth mode drift.
