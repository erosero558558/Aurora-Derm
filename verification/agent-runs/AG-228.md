# AG-228 - Alinear login admin legacy en checklist production

- Fecha: 2026-03-13
- Owner: Ernesto
- Executor: codex

## Resumen

Se alineo `tests/checklist-production.spec.js` con el gate admin canonico.
El caso `2. Panel Admin - Login exitoso y navegacion` ya no usa los
selectores genericos `input[type="password"]` y `button[type="submit"]`;
ahora valida y opera sobre `#adminPassword` y `#loginBtn`, igual que el resto
de los specs recientes del frente admin.

## Archivos

- `tests/checklist-production.spec.js`

## Verificacion

- `npx eslint tests/checklist-production.spec.js`
    - Resultado: sin errores; persistieron warnings heredados de Playwright por
      condicionales y `waitForTimeout` ya existentes en el archivo.
- `$env:TEST_LOCAL_SERVER_PORT='8044'; npx playwright test tests/checklist-production.spec.js --workers=1`
    - Resultado: `10 passed`, `1 skipped`
    - El skip esperado corresponde al caso de login exitoso cuando
      `PIELARMONIA_ADMIN_PASSWORD` no esta definido en este entorno.

## Resultado

Quedo eliminada la deuda latente de selectores genericos en el checklist de
produccion para el gate admin legacy.
