# AG-138 - Admin sony_v3 canary flag contract

Fecha: 2026-03-01
Owner: codex_backend_ops
Dominio: deploy

## Objetivo

Extender el contrato operativo para que `sony_v3` tenga canary real por feature flag backend y no dependa solo de query/localStorage.

## Cambio aplicado

- Se agrego `admin_sony_ui_v3` al default contract de `FeatureFlags` en `lib/features.php` con valor conservador `false`.
- Se agrego `admin_sony_ui_v3` al endpoint estable `/api.php?resource=features` en `controllers/SystemController.php`.
- Se actualizo `GATE-ADMIN-ROLLOUT.ps1` para validar `admin_sony_ui` y `admin_sony_ui_v3` por etapa (`internal`, `canary`, `general`, `rollback`) y para chequear tambien la URL `admin_ui=sony_v3`.
- Se actualizo `ADMIN-UI-CONTINGENCIA.ps1` para mostrar ambos flags, distinguir rollback a v2 vs legacy y documentar el cutover de v3.
- Se actualizaron las pruebas PHP del contrato de flags y del endpoint.

## Validacion ejecutada

1. `php tests/test_features_class.php`
2. `php tests/test-features.php`
3. `php tests/test_system_features_endpoint.php`
4. Parse PowerShell de `GATE-ADMIN-ROLLOUT.ps1`
5. Parse PowerShell de `ADMIN-UI-CONTINGENCIA.ps1`

## Resultado

- Todas las pruebas PHP pasaron.
- Ambos scripts PowerShell parsean sin errores.
- El endpoint de `features` ahora expone `admin_sony_ui_v3` como boolean estable.

## Riesgos residuales

- El flip efectivo de canary sigue siendo externo al repo: `FEATURE_ADMIN_SONY_UI_V3=true` o storage equivalente.
- Falta cerrar el bloque frontend asociado: smoke/runtime/docs y commit limpio de rollout.
