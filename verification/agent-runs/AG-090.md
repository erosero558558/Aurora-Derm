# AG-090 - Frontend public: fuente canonica unica para service pages acne/laser

## Resumen

Se implemento un bloque grande de hardening frontend para evitar drift entre `servicios/acne.html` y `servicios/laser.html`.

Cambios:

- Nuevo generador canonico `servicios/build-service-pages.js`:
    - usa `servicios/acne.html` como fuente base.
    - genera `servicios/laser.html` con transformaciones controladas.
    - soporta modo `--check` para detectar drift sin escribir.
- Integracion en scripts npm:
    - `services:build`
    - `services:check`
    - `build` ahora incluye `services:build`.
    - `test:frontend:qa:services` ahora ejecuta `services:check` antes de Playwright.
- `servicios/laser.html` regenerado desde la fuente canonica.

## Archivos principales

- `servicios/build-service-pages.js`
- `servicios/laser.html`
- `package.json`

## Validacion local

1. `node servicios/build-service-pages.js --check` -> OK
2. `npm run services:check` -> OK
3. `npx eslint servicios/build-service-pages.js` -> OK
4. `npx playwright test tests/version-consistency.spec.js tests/script-version-parity.spec.js tests/deferred-styles-version.spec.js` -> OK (`10 passed`)
5. `npm run test:frontend:qa:services` -> OK (`5 passed`)

## Nota operativa

Hubo una corrida concurrente inicial con `ERR_CONNECTION_REFUSED` por contencion del web server local al ejecutar suites en paralelo; rerun secuencial del mismo scope paso en verde.

## Resultado

Service pages quedan con fuente canonica unica y guardrail automatico de sync/check para prevenir regresiones de paridad entre acne y laser.
