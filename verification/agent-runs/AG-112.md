# AG-112 - Turnero sala P: consola de contingencia kiosco+TV (gestionar pendientes/snapshot)

## Objetivo

Dar control operativo explícito cuando hay degradación de red/backend, sin detener flujo de sala: gestionar pendientes offline en kiosco y administrar respaldo local en TV.

## Cambios implementados

- `src/apps/queue-kiosk/index.js`
    - Consola de contingencia de outbox:
        - resumen de pendientes,
        - listado de solicitudes en cola,
        - acciones `Sincronizar pendientes`, `Descartar mas antiguo`, `Limpiar pendientes`.
    - Atajos nuevos:
        - `Alt+Shift+Y` sincroniza outbox,
        - `Alt+Shift+K` limpia outbox.
    - Hardening de estado de botones durante flush y render continuo de indicadores.
- `src/apps/queue-display/index.js`
    - Controles de respaldo local:
        - `displaySnapshotHint` con edad del snapshot,
        - boton `Limpiar respaldo`.
    - Atajo nuevo `Alt+Shift+X` para limpiar snapshot local.
    - Si se limpia respaldo sin conexion en vivo, la UI cae a estado vacío explícito (`Sin respaldo local disponible`).
- Bundles regenerados:
    - `js/queue-kiosk.js`
    - `js/queue-display.js`
- QA Playwright ampliado:
    - `tests/queue-kiosk.spec.js`
        - verifica consola de outbox (sync + limpieza).
    - `tests/queue-display.spec.js`
        - verifica limpieza de snapshot desde controles de contingencia.

## Validacion ejecutada

- `npx eslint src/apps/queue-kiosk/index.js src/apps/queue-display/index.js tests/queue-kiosk.spec.js tests/queue-display.spec.js` (ok)
- `npm run build` (ok)
- `npx playwright test tests/queue-kiosk.spec.js tests/queue-display.spec.js --reporter=line` (ok, 10 passed)

## Resultado

Bloque grande cerrado con controles operativos completos para contingencia: el kiosco permite operar y gobernar pendientes offline desde UI, y la TV permite gestionar explícitamente el respaldo local cuando backend no está disponible.
