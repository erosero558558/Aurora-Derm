# AG-115 - Stress multi-consultorio C1/C2 (admin + TV)

## Objetivo
Validar y estabilizar escenarios de llamados paralelos por consultorio para evitar duplicados y asegurar consistencia entre Admin y pantalla de sala.

## Cambios implementados
- `tests/admin-queue.spec.js`
  - Se agregaron pruebas de rafaga/doble click, retry tras error transitorio y llamados paralelos C1/C2.
  - Para paralelo C1/C2 se usa disparo atomico de eventos click via `page.evaluate` para evitar serializacion no deseada del runner.
- `tests/queue-integrated-flow.spec.js`
  - Se agrego escenario integrado de llamados paralelos C1/C2 con kiosco + admin + TV.
  - Se ajusto el disparo paralelo con `adminPage.evaluate` para ejecutar ambos clicks en el mismo turno.

## Validacion ejecutada
- `npx eslint tests/admin-queue.spec.js tests/queue-integrated-flow.spec.js`
- `npx playwright test tests/admin-queue.spec.js tests/queue-integrated-flow.spec.js --reporter=line`

## Resultado
- ESLint: OK
- Playwright: `11 passed` (bloque verde)

## Riesgo residual
- Bajo: los escenarios de concurrencia UI quedan cubiertos por pruebas dedicadas de admin e integradas kiosco/admin/TV.
