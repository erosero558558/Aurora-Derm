# AG-076 — Frontend QA Admin: reducir flakiness y warnings lint en specs clave

## Resumen
Se robustecieron tests de Admin para evitar esperas ciegas y condicionales dentro de bloques `test(...)`, reduciendo riesgo de flakiness y alineando la suite con reglas de lint Playwright.

## Archivos modificados
- tests/admin.spec.js
- tests/admin-availability-responsive.spec.js

## Cambios aplicados
- `tests/admin.spec.js`
  - Eliminados `waitForTimeout(...)` en pruebas de login.
  - Reemplazadas ramas condicionales por expectativas directas (`toBeVisible`, estado de `#adminDashboard`).
  - Prueba de contraseña incorrecta ahora valida feedback visible sin lógica condicional.
- `tests/admin-availability-responsive.spec.js`
  - Extraída lógica repetida a helper `ensureAtLeastOneSelectedSlot(...)`.
  - Eliminados condicionales dentro de tests para cumplir `playwright/no-conditional-in-test`.
  - Mantenida cobertura funcional de disponibilidad local/google y acciones copiar/duplicar/limpiar/pegar.

## Validación ejecutada
- `npx eslint tests/admin.spec.js tests/admin-availability-responsive.spec.js tests/admin-navigation-responsive.spec.js tests/admin-quick-nav.spec.js`
  - Resultado: sin warnings en esos archivos.
- `npx playwright test tests/admin.spec.js tests/admin-availability-responsive.spec.js tests/admin-navigation-responsive.spec.js tests/admin-quick-nav.spec.js`
  - Resultado: **19/19** pruebas en verde.

## Riesgo
- Bajo (solo QA frontend, sin cambios de runtime productivo ni APIs).
