# AG-108 - Turnero sala L: hardening realtime kiosco+TV (watchdog + health state)

## Objetivo

Endurecer sincronizacion realtime de kiosco y pantalla de sala para detectar datos estancados y diferenciar mejor estados de reconexion.

## Cambios implementados

- `src/apps/queue-kiosk/index.js`
    - Se agrego watchdog de frescura (`QUEUE_STALE_THRESHOLD_MS = 30000`) para detectar cola estancada por `updatedAt`.
    - `refreshQueueState()` ahora devuelve estado estructurado (`ok/stale/reason`) en vez de booleano simple.
    - `runQueuePollingTick()` ahora separa tres casos:
        - sync saludable (`live`)
        - sync estancada (`reconnecting` por watchdog)
        - error de fetch (`reconnecting` con retry en segundos)
    - Se reforzo el flujo post-checkin/walk-in para dejar mensaje de sincronizacion pendiente si la cola no se refresca.
- `src/apps/queue-display/index.js`
    - Misma estrategia de watchdog (`POLL_STALE_THRESHOLD_MS = 30000`) aplicada a la pantalla TV.
    - `refreshDisplayState()` devuelve estado estructurado.
    - `runDisplayPollTick()` diferencia `live` vs `watchdog stale` vs `fetch_error`, con mensaje de retry.
- Bundles regenerados:
    - `js/queue-kiosk.js`
    - `js/queue-display.js`

## Validacion ejecutada

- `npx eslint src/apps/queue-kiosk/index.js src/apps/queue-display/index.js` (ok)
- `npm run build` (ok)
- `npx playwright test tests/queue-kiosk.spec.js tests/queue-display.spec.js --reporter=line` (ok, 2 passed)

## Resultado

Bloque cerrado con mejor resiliencia operacional en kiosco y TV: ahora el sistema distingue claramente entre desconexion real, reconexion y datos estancados (watchdog), reduciendo falsos "en vivo" cuando la cola no se actualiza.
