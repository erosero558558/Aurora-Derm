# AG-161 — Tighten internal V6 render contracts

## Scope
- Remove silent render fallbacks from internal V6 surfaces where content is already explicit.
- Stop generating telemedicine media and initiative fallbacks in component runtime.
- Filter page-level/internal navigation items so empty labels do not mount into DOM.

## Files
- `src/apps/astro/src/components/public-v6/PageHeadV6.astro`
- `src/apps/astro/src/components/public-v6/ServiceDetailV6.astro`
- `src/apps/astro/src/components/public-v6/ServiceHubV6.astro`
- `src/apps/astro/src/components/public-v6/TelemedicinePageV6.astro`
- `src/apps/astro/src/components/public-v6/LegalPageV6.astro`
- `content/public-v6/es/telemedicine.json`
- `content/public-v6/en/telemedicine.json`
- `verification/public-v6-canonical/artifact-drift.json`

## Changes
1. `PageHeadV6` now filters breadcrumbs and page-menu items before render, avoids empty section-title placeholders, and suppresses the locale divider when no switch link exists.
2. `ServiceDetailV6` now filters section-menu items, derives FAQ cards only from explicit question+answer pairs, and stops rendering empty lead/rail/category/summary nodes.
3. `ServiceHubV6` now filters page-menu entries, removes empty category/CTA wrappers, and uses explicit alt text/title values from content.
4. `TelemedicinePageV6` no longer fabricates initiatives from blocks or media from hardcoded fallback arrays; tele blocks now read explicit `image` from content.
5. `LegalPageV6` now builds page-menu items only from real labels and stops falling back to page-wide hero media for policy cards.
6. `content/public-v6/es|en/telemedicine.json` gained explicit block images so tele blocks remain deterministic without component-side fallback generation.
