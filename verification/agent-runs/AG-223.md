# AG-223 - Align deferred shell fallback spec to public V6 static contract

## Summary

- Replaced the legacy `public-v3` shell expectations in `tests/deferred-shell-static-fallback.spec.js` with the current `public-v6` static fallback contract.
- The spec now aborts `public-v6-shell.js` and verifies the delivered HTML remains usable without runtime hydration:
    - `data-v6-header`
    - `data-v6-hero`
    - `data-v6-news-strip`
    - `data-v6-editorial`
    - `data-v6-corporate-matrix`
    - `data-v6-booking-status`
- Added explicit guards that the removed legacy shell does not reappear (`data-public-nav`, `data-stage-carousel`, `data-program-grid`, `data-booking-bridge-band`, `appointmentForm`).

## Reproduced issue

- `npx playwright test tests/deferred-shell-static-fallback.spec.js --workers=1`
- Original failure timed out waiting for `[data-public-nav]`, which no longer exists on `/es/` after the `public-v6` migration.

## Verification

- `npx eslint tests/deferred-shell-static-fallback.spec.js`
- `TEST_LOCAL_SERVER_PORT=8031 npx playwright test tests/deferred-shell-static-fallback.spec.js --workers=1`

## Outcome

- `1/1` Playwright test passing on the updated contract.
- ESLint clean for the updated spec.
