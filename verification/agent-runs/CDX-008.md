# CDX-008 - Telemedicine ops hardening: health snapshot, metrics and admin triage

## Summary

Se agrego un snapshot operacional reusable de telemedicina para exponer:

- `checks.telemedicine` en `health` sin PHI
- `data.telemedicineMeta` en `/data` con cola de revision para staff
- gauges Prometheus de telemedicina en `/metrics`

El cambio es backend-only y no altera el contrato publico legacy de
`appointments`, `payment-intent` ni `transfer-proof`.

- `controllers/HealthController.php`
- `controllers/AdminDataController.php`
- `controllers/SystemController.php`

- `tests/Integration/TelemedicineAdminReadModelTest.php`
- `tests/Integration/TelemedicineMetricsExportTest.php`

## Behaviour

- `health` resume intakes, media, integrity y `reviewQueueCount`.
- `health` no expone `reviewQueue` ni datos de paciente.
- `/data` agrega `telemedicineMeta.summary` y `telemedicineMeta.reviewQueue`.
- `/metrics` exporta conteos por status, suitability, channel, media kind y
  storage mode, mas drift operacional (`unlinked_intakes`, `staged_legacy`,
  etc.).
