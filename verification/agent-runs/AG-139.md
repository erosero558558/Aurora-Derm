# AG-139 - Admin sony_v3 canary smoke docs and clean commit

Fecha: 2026-03-01
Owner: codex_backend_ops
Dominio: frontend-admin

## Objetivo

Cerrar el canary de `sony_v3` con smoke real, gate operativo y documentacion alineada al nuevo contrato de flags.

## Cambio aplicado

- `tests/admin-ui-runtime-smoke.spec.js` ahora exige `admin_sony_ui_v3` como boolean estable desde `/api.php?resource=features`.
- `tests/admin-ui-variant.spec.js` actualiza el mock por defecto para reflejar el contrato real (`admin_sony_ui_v3=false` cuando no se explicita override).
- `docs/ADMIN-UI-ROLLOUT.md` se alineo al nuevo modelo: `admin_sony_ui_v3` ya no es ausencia tolerada sino flag booleana con default conservador `false`.
- `docs/DEPLOYMENT.md` documenta canary de `sony_v3` con `admin_sony_ui=true` y `admin_sony_ui_v3` por env/storage/rollout porcentual.

## Validacion ejecutada

1. `npx playwright test tests/admin-ui-runtime-smoke.spec.js tests/admin-ui-variant.spec.js --workers=1`
2. `powershell -NoProfile -ExecutionPolicy Bypass -File .\GATE-ADMIN-ROLLOUT.ps1 -Domain http://127.0.0.1:8010 -Stage canary`
    - ejecutado contra PHP local con:
    - `FEATURE_ADMIN_SONY_UI=true`
    - `FEATURE_ADMIN_SONY_UI_V3=true`
3. `npm run test:frontend:qa:admin`

## Resultado

- Runtime smoke y variant loader: verde.
- Gate canary: verde con `admin_sony_ui=true` y `admin_sony_ui_v3=true`.
- QA admin amplia: `50 passed`, `1 skipped`.

## Riesgos residuales

- El canary efectivo en un entorno real sigue requiriendo flip externo de `FEATURE_ADMIN_SONY_UI_V3=true` o storage equivalente.
- El commit debe excluir cambios ajenos no relacionados (`js/public-v2-shell.js`).
