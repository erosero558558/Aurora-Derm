# AG-111 - Turnero sala O: continuidad offline kiosco + snapshot de respaldo en TV

## Objetivo

Aumentar continuidad operativa del turnero cuando hay caidas intermitentes o ausencia temporal de backend, sin detener flujo de sala.

## Cambios implementados

- `src/apps/queue-kiosk/index.js`
    - Outbox offline local para `queue-ticket` y `queue-checkin`.
    - Guardado en `localStorage` (`queueKioskOfflineOutbox`) con limite y metadatos de intentos.
    - Reintento automatico al reconectar (`online`) y durante polling/manual refresh.
    - Render de ticket pendiente offline con codigo local (`PEND-xx`) y estado visible.
    - Hint operativo dedicado de pendientes (`#queueOutboxHint`) con conteo/antiguedad.
- `src/apps/queue-display/index.js`
    - Persistencia de ultimo snapshot valido de cola (`queueDisplayLastSnapshot`).
    - Restore automatico del snapshot al iniciar y ante fallos de `queue-state`.
    - Mensajeria operativa explicita cuando se usa respaldo local (sin backend).
- Bundles regenerados:
    - `js/queue-kiosk.js`
    - `js/queue-display.js`
- QA Playwright ampliado:
    - `tests/queue-kiosk.spec.js`
        - nuevo escenario de outbox offline + sincronizacion al reconectar.
    - `tests/queue-display.spec.js`
        - nuevo escenario de snapshot local cuando backend no responde.

## Validacion ejecutada

- `npx eslint src/apps/queue-kiosk/index.js src/apps/queue-display/index.js tests/queue-kiosk.spec.js tests/queue-display.spec.js` (ok)
- `npm run build` (ok)
- `npx playwright test tests/queue-kiosk.spec.js tests/queue-display.spec.js --reporter=line` (ok, 8 passed)

## Resultado

Turnero robustecido para continuidad de sala: kiosco ya no pierde solicitudes por caidas de red y TV mantiene ultimo estado operativo visible mientras reconecta backend.
