# AG-079 — Frontend admin UX: command bar + contextual actions + refresh status

## Objetivo

Entregar un bloque grande de UX para Admin con mejoras visibles de productividad y navegación sin tocar backend:

1. Barra de comando rápido (`Ctrl+K`, `Enter`, acciones por texto).
2. Acciones contextuales por sección (dashboard/citas/callbacks/reseñas/disponibilidad).
3. Estado de frescura de datos + botón de actualización manual (`Alt+Shift+R`).

## Cambios implementados

- `admin.html`
    - Nueva franja `admin-productivity-strip` debajo del header.
    - Input `#adminQuickCommand`, botón `run-admin-command`, panel `#adminContextActions`.
    - Estado de refresh `#adminRefreshStatus` y botón `refresh-admin-data`.
    - Versionado de assets a `admin-20260226-nav6`.

- `admin.css`
    - Estilos completos para la franja de productividad.
    - Estados visuales del refresh (`status-pill-live`, `status-pill-stale`).
    - Soporte responsive (tablet/móvil) y dark mode para nuevos componentes.

- `src/apps/admin/index.js`
    - Parser de comandos rápidos (ej.: `citas hoy`, `callbacks pendientes`, `disponibilidad hoy`, `exportar csv`, `actualizar`, `ayuda`).
    - Mapa de acciones contextuales por sección y render dinámico.
    - Shortcuts nuevos:
        - `Ctrl+K`/`Cmd+K`: enfoca comando rápido.
        - `Alt+Shift+R`: refresca datos y render de sección activa.
        - `/`: fallback a comando rápido fuera de campos de edición.
    - Estado temporal de refresh con ticker y clasificación fresh/stale.
    - Integración con acciones existentes sin romper hashes o quick nav.

- `tests/admin-quick-nav.spec.js`
    - Nueva prueba E2E para `Ctrl+K` + ejecución de comando contextual.
