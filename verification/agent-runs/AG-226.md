# AG-226 - Realign production checklist E2E to public V6 booking fallback

## Summary

- Kept the public `checklist-production` coverage aligned with the current `public-v6` fallback flow already present in the spec.
- Reworked the admin access assertion so the unauthenticated gate works with both admin login modes: `legacy_password` and `openclaw_chatgpt`.
- Replaced the brittle `input[type="password"]` assumption with mode-aware checks against the current admin shell (`#adminPassword`, `#openclawLoginStage`, `#adminLoginStatusTitle`, `#adminLoginStatusMessage`).

## Reproduced issue

```powershell
$env:TEST_LOCAL_SERVER_PORT='8040'; npx playwright test tests/checklist-production.spec.js --workers=1
```

Observed failure before the fix:

- `2. Panel Admin - Login fallido` timed out trying to fill `input[type="password"]`.
- The current admin shell had switched to the OpenClaw access gate, so the password input existed in the DOM but was not the visible login surface.

## Verification

```powershell
npx eslint tests/checklist-production.spec.js
$env:TEST_LOCAL_SERVER_PORT='8041'; npx playwright test tests/checklist-production.spec.js --workers=1
```

Notes:

- ESLint finished with warnings only, all pre-existing `playwright/*` warnings in this long-lived checklist spec; no new lint errors were introduced.

## Outcome

- `10/10` checklist tests passing, with `1` expected skip for the successful admin login path when `PIELARMONIA_ADMIN_PASSWORD` is not provided.
- The admin unauthenticated gate now validates the real runtime contract instead of assuming the legacy password form is always the visible entrypoint.
