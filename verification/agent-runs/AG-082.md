# AG-082 � Frontend admin UX: callbacks triage rail + context action hardening

## Objetivo

Implementar un bloque grande de UX Admin para callbacks, reduciendo friccion operativa y eliminando colisiones de acciones contextuales que afectaban QA.

## Cambios aplicados

- `admin.html`
    - Nuevo panel `#callbacksOpsPanel` en seccion callbacks:
        - salud de cola (`#callbacksOpsQueueHealth`)
        - metricas (`#callbacksOpsPendingCount`, `#callbacksOpsUrgentCount`, `#callbacksOpsTodayCount`)
        - siguiente contacto (`#callbacksOpsNext`)
        - accion primaria (`#callbacksOpsNextBtn`, `data-action="callbacks-triage-next"`)
    - Versionado de assets a `admin-20260226-nav8`.
- `admin.css`
    - Estilos nuevos para panel de triage callbacks (`.callbacks-ops-*`) con soporte responsive y dark mode.
- `src/apps/admin/index.js`
    - Endurecimiento de acciones contextuales para disponibilidad con namespace:
        - `context-availability-today`
        - `context-availability-next`
        - `context-copy-availability-day`
    - Nueva accion contextual callbacks:
        - `context-open-callbacks-next`
    - Nuevo handler:
        - `callbacks-triage-next`.
- `src/apps/admin/modules/callbacks.js`
    - Motor de triage:
        - calculo de antiguedad en cola
        - severidad (`estable` / `atencion` / `prioridad alta`)
        - seleccion de siguiente contacto sugerido
    - Render del panel operacional callbacks.
    - Export de helper `focusNextPendingCallback()` para llevar foco operativo.
    - Atributos `data-*` en callback cards para soporte de triage.
- `tests/admin-availability-responsive.spec.js`
    - Selectores scopeados a `#availabilityDayActions` para evitar colision por acciones duplicadas en contexto.
- `tests/admin.spec.js`
    - Nuevo test de integracion de triage callbacks.
