# AG-102 - Turnero sala F: hardening UX admin para liberar y consultorio ocupado

Fecha: 2026-02-26
Owner: Ernesto
Executor: codex

## Cambios implementados

- `src/apps/admin/modules/queue.js`
    - Nuevo guardrail UI de acciones por estado:
        - `Liberar` visible para tickets en estado `called`.
        - Acciones terminales (`completar/no_show/cancelar`) ocultas para tickets ya terminales.
        - Reasignacion C1/C2 deshabilitada en estados terminales.
    - Manejo operacional de errores de consultorio ocupado:
        - Si `queue-call-next` o `queue-ticket` devuelve error de consultorio ocupado, se refresca estado de cola en background.
        - Se muestra warning accionable al operador (en vez de error generico).
    - Mensajes de exito especificos por accion (`re-llamar`, `liberar`, `completar`, `no_show`, `cancelar`, `reasignar`).

## Validacion

- `npm run build` -> OK
- `npx eslint src/apps/admin/modules/queue.js` -> OK
- `npx playwright test tests/admin-queue.spec.js` -> OK

## Resultado

Operacion de recepcion mas robusta: el admin puede liberar turnos llamados, evita acciones inconsistentes por estado y recibe feedback claro cuando un consultorio ya esta ocupado.
