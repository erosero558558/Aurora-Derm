# AG-117 - Astro public redesign sony-style ES/EN routing

## Objetivo
Completar el rediseño público en Astro con rutas canónicas bilingües (`/es/*`, `/en/*`), migración 301 de rutas legacy, y gates de calidad (QA + Lighthouse + SEO técnico) sin romper booking/chat.

## Cambios implementados
- Rediseño público Astro con páginas:
  - Home ES/EN (`/es/`, `/en/`)
  - Telemedicina (`/es/telemedicina/`, `/en/telemedicine/`)
  - Servicios (`/es/servicios/:slug/`, `/en/services/:slug/`)
  - Legales (`/es/legal/*`, `/en/legal/*`)
- Migración y compatibilidad:
  - Mapa 301 en Apache (`.htaccess`) y Nginx (`nginx-pielarmonia.conf`) para legacy público (`/`, `/index.html`, `/*.html`, `/servicios/*.html`, `/ninos/*.html`).
  - Preservación de query params en redirects Nginx (`$is_args$args`).
- Runtime config público (aditivo, no breaking):
  - `GET /api.php?resource=public-runtime-config` en backend (`SystemController`, `lib/routes.php`, `lib/ApiConfig.php`).
  - Bridge frontend para fallback de configuración sin bloquear render.
- Hardening de calidad frontend:
  - Runner Lighthouse premium funcional en Windows forzado (`LIGHTHOUSE_FORCE_WINDOWS=1`).
  - Corrección de accesibilidad en booking form (`label` + `for`/`id`) para cumplir umbral de A11y.
  - Corrección de switch de idioma para mantener ruta equivalente ES/EN.
  - Captura baseline visual desktop/mobile con manifiesto reproducible (`bin/capture-public-baseline.js`) e integración como artefacto en `frontend-premium-qa`.
  - Corrección de paridad de hints de reserva por servicio ES/EN usando `cta.service_hint` del catálogo (`src/apps/astro/src/lib/content.js`), evitando CTAs con `?service=` no compatible con `#serviceSelect`.
- Cobertura de QA agregada:
  - `tests/public-routing-redirects.spec.js`
  - `tests/language-route-equivalence.spec.js`
  - `tests/public-seo-routing.spec.js` (canonical/hreflang/sitemap/robots)
  - `tests/service-booking-hints-parity.spec.js` (CTA embebido hacia `#citas` + preselección real en `/es/` y `/en/`)
  - Integración del nuevo spec en `test:frontend:qa:premium`.
- Alineación de contratos legacy al shell `Public V2`:
  - `tests/deferred-content-fallback.spec.js` migra de `#servicios` al shell editorial nuevo (`[data-stage-carousel]`, `[data-program-grid]`, `[data-featured-story]`) manteniendo la exigencia de booking bridge visible.
  - `tests/deferred-shell-static-fallback.spec.js` valida que la home `/es/` mantenga `public-nav`, shell editorial y `#appointmentForm` aun cuando falle el runtime legacy.
  - `tests/home-deferred-hydration.spec.js` valida boot del bridge legacy sobre la shell `Public V2` usando `document.documentElement.dataset.publicV2RuntimeBooted`.
  - `tests/service-booking-hints-parity.spec.js` deja de asumir redirect legacy a home y valida continuidad real del bridge embebido en fichas de servicio.
  - `src/apps/astro/src/components/JourneyRail.astro` deja de emitir `#servicios` y usa la ruta canónica del hub (`/es/servicios/`, `/en/services/`) con `ctaTarget=service_hub`.
  - `js/service-navigation-analytics.js` clasifica superficies editoriales `Public V2`, preserva `open_public_cta` / `start_booking_from_service` y ajusta el offset de scroll para `.public-nav`.
- Hardening de release/cutover en deploy:
  - Nuevo smoke runtime de routing público: `bin/check-public-routing-smoke.js`.
  - Valida canónicas ES/EN en vivo (2xx) y redirects legacy (301) preservando query params.
  - Nuevo smoke runtime de conversión pública: `bin/check-public-conversion-smoke.js`.
  - Valida hooks y CTAs críticos de conversión (`#citas`, `#appointmentForm`, `#serviceSelect`, `#chatbotWidget`, `start_booking_from_service`, CTAs telemedicina y retorno legal), además de contrato `public-runtime-config`.
  - Integrado en `.github/workflows/deploy-staging.yml` post-deploy.
  - Integrado en `.github/workflows/deploy-hosting.yml` para canary (staging) y producción.
  - Contratos de workflow y script en:
    - `tests-node/public-conversion-smoke.test.js`
    - `tests-node/public-routing-smoke.test.js`
    - `tests-node/public-routing-deploy-workflows-contract.test.js`
