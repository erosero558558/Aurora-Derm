# CDX-041 - Admin shell core daily-use: auth and readiness lane

Estado: `done`
Inicio: `2026-03-14`
Estrategia: `STRAT-2026-03-admin-operativo`
Subfrente: `SF-backend-admin-operativo`

## Objetivo

- Dejar `OpenClaw` como acceso primario del admin shell core.
- Mantener la contingencia web `clave + 2FA` solo cuando el backend la anuncie.
- Sostener `dashboard`, `agenda`, `pendientes`, `horarios` e `historia clinica` via `/data` e `internalConsoleMeta` incluso con bloqueos clinicos.

## Archivos reservados

- `admin-auth.php`
- `controllers/AdminDataController.php`
- `lib/InternalConsoleReadiness.php`

- `tests/test_admin_auth_openclaw_facade.php`
- `tests/test_operator_auth.php`
- `tests/Integration/OperatorAuthControllerTest.php`
- `tests/Integration/ClinicalHistoryAdminReadModelTest.php`

## Contrato validado

- `admin-auth.php?action=status|start|login|logout` mantiene `OpenClaw` como primario y solo expone contingencia `clave + 2FA` cuando el backend la anuncia.
- `/api.php?resource=data` e `internalConsoleMeta` sostienen el shell core aun cuando la parte clinica no esta lista.
- No se agregaron endpoints nuevos; se cerraron contratos sobre superficies ya existentes.
