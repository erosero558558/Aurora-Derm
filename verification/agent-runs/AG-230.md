# AG-230 - Expandir helper auth a specs admin medianos

- Fecha: 2026-03-13
- Owner: Ernesto
- Executor: codex

## Resumen

Se expandio el helper compartido de `admin-auth.php` a una segunda tanda de
specs admin medianos: appointments, availability, callbacks y navigation
responsive. Con esto esos tests ya no dependen de un payload auth incompleto y
quedan alineados con el contrato canónico del gate admin.

## Archivos

- `tests/admin-appointments-quick-filters.spec.js`
- `tests/admin-appointments-responsive.spec.js`
- `tests/admin-availability-readonly.spec.js`
- `tests/admin-availability-responsive.spec.js`
- `tests/admin-callbacks-triage.spec.js`
- `tests/admin-navigation-responsive.spec.js`

## Verificacion

- `npx eslint tests/admin-appointments-quick-filters.spec.js tests/admin-appointments-responsive.spec.js tests/admin-availability-readonly.spec.js tests/admin-availability-responsive.spec.js tests/admin-callbacks-triage.spec.js tests/admin-navigation-responsive.spec.js`
    - Resultado: sin errores. Persistieron 2 warnings heredados en `tests/admin-availability-responsive.spec.js` por `playwright/prefer-to-have-length`.
- `$env:TEST_LOCAL_SERVER_PORT='8048'; npx playwright test tests/admin-appointments-quick-filters.spec.js tests/admin-appointments-responsive.spec.js tests/admin-availability-readonly.spec.js tests/admin-availability-responsive.spec.js tests/admin-callbacks-triage.spec.js tests/admin-navigation-responsive.spec.js --workers=1`
    - Resultado: `19 passed`

## Resultado

Quedo ampliada la cobertura del helper auth en la familia admin. La siguiente
deuda homogenea visible queda concentrada sobre todo en `tests/admin-queue.spec.js`
y en algunos specs admin restantes que aun mockean solo `authenticated/csrfToken`.
