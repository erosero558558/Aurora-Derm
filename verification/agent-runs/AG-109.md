# AG-109 - Turnero sala M: UX degradada kiosco+TV (fallback visible + refresh manual)

## Objetivo

Mejorar operacion en sala cuando la cola entra en estado degradado, con mensajes accionables y recuperacion manual inmediata desde kiosco y TV.

## Cambios implementados

- `src/apps/queue-kiosk/index.js`
    - Se agrego hint operativo dinamico (`queueOpsHint`) con estado de salud y acciones recomendadas.
    - Se agrego boton de recuperacion manual (`queueManualRefreshBtn`) y atajo `Alt+Shift+R`.
    - Polling ahora comunica claramente: estable, degradado por watchdog, reconexion y offline.
    - Se registran mensajes de continuidad operativa cuando check-in/walk-in se crea pero la sincronizacion falla.
- `src/apps/queue-display/index.js`
    - Se agrego hint operativo (`displayOpsHint`) para personal en sala.
    - Se agrego boton `displayManualRefreshBtn` + atajo `Alt+Shift+R`.
    - Polling y refresh manual separan estado estable vs datos estancados vs reconexion.
- Bundles regenerados:
    - `js/queue-kiosk.js`
    - `js/queue-display.js`
- QA:
    - `tests/queue-kiosk.spec.js`: nuevo escenario watchdog + recuperacion manual.
    - `tests/queue-display.spec.js`: nuevo escenario watchdog + recuperacion manual.

## Validacion ejecutada

- `npx eslint src/apps/queue-kiosk/index.js src/apps/queue-display/index.js tests/queue-kiosk.spec.js tests/queue-display.spec.js` (ok)
- `npm run build` (ok)
- `npx playwright test tests/queue-kiosk.spec.js tests/queue-display.spec.js --reporter=line` (ok, 4 passed)

## Resultado

Bloque grande cerrado con UX de contingencia visible y operable en kiosco/TV, reduciendo ambigüedad cuando la cola pierde frescura o conectividad.
