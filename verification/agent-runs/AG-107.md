# AG-107 - Turnero sala K: consola operativa admin (historial + watchdog + atajos de estado)

## Objetivo

Cerrar un bloque grande de hardening en el modulo de turnero admin con trazabilidad operativa en pantalla y deteccion temprana de cola estancada.

## Cambios implementados

- `src/apps/admin/modules/queue.js`
    - Se agrego consola de historial operativo dentro de la seccion de turnero (`queueActivityPanel`) con ultimos eventos y estado de sincronizacion.
    - Se incorporo registro estructurado de eventos para:
        - llamado por consultorio
        - acciones de ticket
        - acciones masivas
        - reimpresion
        - cambios de estado de sincronizacion realtime/manual
    - Se agrego watchdog de frescura de cola (`QUEUE_STALE_THRESHOLD_MS = 30000`) para marcar `reconnecting` cuando `queueMeta.updatedAt` queda estancado.
    - Se estandarizo el manejo de estado sync con `updateQueueSyncState(...)` para reducir estados ambiguos y registrar transiciones relevantes.
    - Se amplio hint de atajos incluyendo filtros de estado.
- `src/apps/admin/index.js`
    - Nuevos atajos de estado en seccion turnero:
        - `Alt+Shift+W` -> `waiting`
        - `Alt+Shift+C` -> `called`
        - `Alt+Shift+A` -> `all`
        - `Alt+Shift+I` -> `walk_in`
- `tests/admin-queue.spec.js`
    - Nuevo test E2E para watchdog estancado + trazabilidad en panel de actividad.
    - Test de atajos extendido para cubrir filtros de estado nuevos.
- `admin.js`
    - Bundle regenerado con cambios de `src/apps/admin/index.js` y `src/apps/admin/modules/queue.js`.
