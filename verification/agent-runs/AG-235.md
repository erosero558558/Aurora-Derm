# AG-235 - Canonicalizar login flow auth mock en admin spec

## Resumen

- Se extrajo a `tests/helpers/admin-auth-mocks.js` el flujo stateful de auth legacy para `status/login/login-2fa/logout`.
- El helper compartido ahora cubre los dos contratos reutilizables de Playwright para `admin-auth.php`: login legacy y login OpenClaw.
- `tests/admin.spec.js` dejo de mantener un `page.route(/\/admin-auth\.php...)` inline y ahora consume `installLegacyAdminLoginFlowMock`.

## Archivos

- `tests/helpers/admin-auth-mocks.js`
- `tests/admin.spec.js`

    - Resultado: `0` errores, `0` warnings.

    - Resultado: `22/22` passing.
3. `Get-ChildItem tests -Recurse -Filter *.spec.js | Select-String -Pattern 'page\.route\(/\\/admin-auth\\.php'`

## Notas

- Con este bloque ya no quedan mocks inline de `admin-auth.php` en `tests/*.spec.js`.
- Las referencias restantes a `admin-auth.php` en Playwright son requests reales (`logout`, status checks o chequeos de entorno), no stubs por `page.route`.
