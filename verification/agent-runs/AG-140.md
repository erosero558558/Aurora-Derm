# AG-140 - Admin sony_v3 section refinement workbenches

## Resumen
- Reemplace los reexports de dashboard, appointments y callbacks por renderers sony_v3 nativos.
- Mantengo el mismo store, filtros, acciones `data-action` y mutaciones API para no romper operacion.
- El cambio se concentro en jerarquia, priorizacion y lectura operativa dentro del shell v3.

## Archivos
- src/apps/admin-v3/sections/dashboard.js
- src/apps/admin-v3/sections/appointments.js
- src/apps/admin-v3/sections/callbacks.js

## Validacion
- npx eslint src/apps/admin-v3/sections/dashboard.js src/apps/admin-v3/sections/appointments.js src/apps/admin-v3/sections/callbacks.js
- npx playwright test tests/admin-v3-shell.spec.js tests/admin-v3-visual.spec.js --workers=1
- npx playwright test tests/admin-ui-runtime-smoke.spec.js --workers=1

## Resultado
- Dashboard v3 ahora resume una sola prioridad operacional, con digest mas claro de agenda, callbacks y funnel.
- Appointments v3 usa spotlight, workbench y tabla con narrativa de prioridad sin depender del renderer v2.
- Callbacks v3 usa cola priorizada, siguiente accion y bulk state propios sin depender del renderer v2.
