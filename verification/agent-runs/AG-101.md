# AG-101 - Turnero sala E: hardening backend de cola y consultorios

Fecha: 2026-02-26
Owner: Ernesto
Executor: codex

## Cambios implementados

- `lib/QueueService.php`
    - Recalculo dinamico de prioridad para tickets de cita en espera (`appt_overdue|appt_current`) usando datos de cita.
    - `callNext` ahora bloquea si el consultorio ya tiene ticket `called` activo (`queue_consultorio_busy`).
    - Nueva accion de ticket `liberar|release` para devolver turno a `waiting` y limpiar `calledAt/consultorio`.
    - Guardrails de transicion:
        - re-llamar sobre estados terminales devuelve `queue_transition_invalid`.
        - evita asignar ticket `called` a consultorio ocupado.
    - Normalizacion de campos de estado al pasar a `waiting|called|terminal`.
- `tests/test_queue_service.php`
    - Nuevos casos de prueba para:
        - bloqueo por consultorio ocupado.
        - accion `liberar`.
        - refresco dinamico de prioridad previo a `queue-state` y `call-next`.

Endurecido el flujo operativo del turnero para evitar llamados simultaneos por consultorio y mantener prioridad de citas consistente en tiempo real.
