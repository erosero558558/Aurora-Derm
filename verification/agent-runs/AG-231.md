# AG-231 - Canonical Windows public cutover on local workspace host

## Resumen

- Fecha: 2026-03-13
- Workspace canonico publicado: `C:\dev\pielarmonia-workspace`
- Dominio publico activo: `https://pielarmonia.com`
- Tunnel activo: `a2067e67-a462-41de-9d43-97cd7df4bda0` (`pielarmonia-local-host`)
- Runtime canonico:
  - `caddy.exe` con [`ops/caddy/Caddyfile`](C:/dev/pielarmonia-workspace/ops/caddy/Caddyfile)
  - `php-cgi.exe` en `127.0.0.1:9000`
  - `cloudflared.exe` hacia `http://127.0.0.1`
  - `operator-auth-bridge.js` en `127.0.0.1:4173`
- Autoarranque:
  - Startup shim: `C:\Users\ernes\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\Pielarmonia Hosting Stack.cmd`
  - La tarea programada no pudo registrarse por permisos del usuario; el startup shim queda como mecanismo efectivo de arranque al iniciar sesion.

## Cambios aplicados

- Se versiono el edge Windows en `ops/caddy/Caddyfile` con:
  - redirects canonicos `/ -> /es/`
  - redirects legacy (`/index.html`, `/telemedicina.html`, `servicios/*`, `ninos/*`)
  - hardening para `env.php`, `docs/`, `scripts/`, `vendor/`, `data/` y artefactos raiz sensibles
  - `php_fastcgi` para endpoints PHP publicos canonicos
  - cache headers para estaticos y `no-store` para endpoints dinamicos
- Se agregaron scripts operativos:
  - `scripts/ops/setup/ARRANCAR-HOSTING-WINDOWS.ps1`
  - `scripts/ops/setup/PARAR-HOSTING-WINDOWS.ps1`
  - `scripts/ops/setup/CONFIGURAR-HOSTING-WINDOWS.ps1`
- Se actualizo `env.php` local para anunciar:
  - `PIELARMONIA_OPERATOR_AUTH_SERVER_BASE_URL=https://pielarmonia.com`
  - `PIELARMONIA_OPERATOR_AUTH_HELPER_BASE_URL=http://127.0.0.1:4173`
  - `FIGO_CHAT_ENDPOINT=http://127.0.0.1/figo-backend.php`
