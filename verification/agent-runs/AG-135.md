# AG-135 — Turnero sala XVII: lock estricto C1/C2 + seguridad numpad en admin

## Objetivo

- Endurecer la operacion del turnero en admin para evitar llamados cruzados por error humano.
- Reforzar acciones sensibles por numpad con confirmacion explicita y cancelacion segura.

## Cambios implementados

- `src/apps/admin/index.js`
    - Lock de estacion aplicado en flujo de llamado con bloqueo real para consultorio opuesto.
    - Guard de accion sensible para `Numpad.` y `Numpad-`:
        - primer paso: prepara accion
        - segundo paso: `NumpadEnter` confirma
        - `Esc` cancela
    - Dialogo accesible de confirmacion sensible inyectado en runtime (`#queueSensitiveConfirmDialog`).
    - Telemetria operativa aditiva: `sensitive_action_prepared`, `action_confirmed`, `action_cancelled`.
    - Sincronizacion de UI de estacion con estado de guard y bloqueo visual/funcional coherente.
- `tests/admin-queue.spec.js`
    - Ajustes de escenarios QA para reflejar el flujo seguro nuevo:
        - bloqueo de consultorio opuesto en modo lock
        - preparacion y confirmacion de acciones sensibles con numpad
        - cross-platform numpad con confirmacion por `NumpadEnter`

## Validacion

- `npx eslint src/apps/admin/index.js tests/admin-queue.spec.js` ✅
- `npm run build` ✅
- `npx playwright test tests/admin-queue.spec.js` ✅ (23/23)

## Resultado

- Operacion por estacion mas segura en C1/C2.
- Menor riesgo de `no_show/completar` accidental por numpad.
- QA alineada al comportamiento estrella del turnero.
