## Manual recovery applied

The server already had an auto-sync job at `/root/sync-pielarmonia.sh`, but it was failing because `deploy-public-v3-live.sh` did not have execute permissions.

Recovery executed on the VPS/OpenClaw host:

1. `chmod +x /var/www/figo/bin/deploy-public-v3-live.sh`
2. `/usr/bin/flock -n /tmp/sync-pielarmonia.lock /root/sync-pielarmonia.sh`
3. `php -r "require '/var/www/figo/lib/features.php'; FeatureFlags::enable('admin_sony_ui'); FeatureFlags::enable('admin_sony_ui_v3'); print_r(FeatureFlags::getAll());"`

The manual sync completed successfully and the host reported:

- `Commit actual: 1202559`
- `Deploy completado en 12025599b1e83797ff5f62d26b3ff26aa5e9388f`
