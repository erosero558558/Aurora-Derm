# AG-240 - Consolidar mocks de apps operativas desktop en admin queue spec

## Resumen

- Se consolidó el setup repetido de las apps operativas desktop dentro de
  `tests/admin-queue.spec.js`.
- El spec ahora usa builders dedicados para instancias desktop del operador y
  para `queueSurfaceStatus`, en lugar de reescribir payloads largos por cada
  escenario del hub operativo.
- Se agregó `installQueueOperationalAppsApiMocks(...)` para reutilizar el
  contrato base de `/data`, `/health`, `/queue-state` y las superficies del
  operador.
- Se migraron los escenarios de configuración local y reintento de apertura a
  ese helper, sin cambiar el comportamiento esperado del hub admin.

## Archivos

- `tests/admin-queue.spec.js`
- `verification/agent-runs/AG-240.md`

    - Resultado: `0` errores, `7` warnings heredados.
- `TEST_LOCAL_SERVER_PORT=8017 npx playwright test tests/admin-queue.spec.js --workers=1`

## Resultado

El bloque de apps operativas desktop quedó bastante más mantenible: los casos
siguen cubriendo configuración local y reintentos del operador, pero ahora lo
hacen sobre builders compartidos en vez de payloads repetidos.
