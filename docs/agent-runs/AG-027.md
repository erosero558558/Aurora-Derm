# AG-027 - Resolver fallo workflow `Post-Deploy Gate (Git Sync)`

## Contexto
- Task: `AG-027`
- Source signal: `workflow:post-deploy-gate-git-sync:main`
- Falla observada: run `22383617882`
- URL run fallido: https://github.com/erosero558558/piel-en-armonia/actions/runs/22383617882

## Causa raiz
El gate fallaba en `Ejecutar gate estricto` por una sola regla bloqueante:
- `index remoto styles.css diferente`
- Diferencia era solo de query version (`?v=`) entre local y remoto durante ventana de hash warning.

Con `SkipAssetHashChecks` activo por ventana temporal, esa diferencia estaba siendo tratada como error bloqueante.

## Cambio aplicado
Se ajusto `VERIFICAR-DESPLIEGUE.ps1`:
- Si `SkipAssetHashChecks=true` y la diferencia en `styles.css` o `styles-deferred.css` es solo de version query (mismo path base), se reporta `WARN` en lugar de `FAIL`.
- Si cambia el path del asset o falta la referencia, sigue siendo `FAIL`.

Commit de fix:
- `27f2b56` - `fix(gate): make css query-version drift advisory during hash warning window`

## Validacion
1. Verificacion local contra produccion:
   - `pwsh -File .\VERIFICAR-DESPLIEGUE.ps1 -Domain https://pielarmonia.com -SkipAssetHashChecks -AssetHashRetryCount 1 -AssetHashRetryDelaySec 2`
   - Resultado: `Resultado: verificacion de despliegue OK.`
2. Run de workflow tras el fix:
   - Run exitoso: `22383950985`
   - URL: https://github.com/erosero558558/piel-en-armonia/actions/runs/22383950985
   - Resultado: `success`

## Resultado
Se cierra AG-027 con evidencia de correccion y run verde del gate.
